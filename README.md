## What We Teach

### ⚙️ DevOps Engineering — 3 Months
The full pipeline: Git → CI/CD → Docker → Kubernetes → Terraform → Monitoring. We don't just show you the commands, we build real pipelines together in class.

**Tools covered:** Git, Jenkins, GitHub Actions, Docker, Kubernetes, Terraform, Ansible, Prometheus, Grafana, AWS/Azure

### ☁️ Cloud Computing — 3 Months
AWS and Azure, both. Starting from what the cloud actually is, ending with you building and deploying a real multi-tier application on it.

**Tools covered:** EC2, S3, RDS, Lambda, VPC, IAM, Azure VMs, Blob Storage, Azure AD, Cost Management

### 🐧 Linux Sysadmin — 2 Months
This is where most DevOps journeys should start but don't. We go from filesystem basics to bash scripting to hardening a production server.

**Tools covered:** Ubuntu/CentOS, Bash, cron, iptables, ufw, systemd, SSH, package managers

### 🌐 Network Engineer — 2 Months
TCP/IP, subnetting, routing protocols, VLANs, VPNs — the stuff that actually runs the internet. Built for people who want CCNA-level understanding without paying CCNA prices.

**Tools covered:** Cisco Packet Tracer, Wireshark, pfSense, OSPF, BGP, IPSec VPN

### 🐍 Python & Java — 2 Months
Programming fundamentals done right. OOP, file handling, automation scripting in Python. Core Java with collections, exceptions, and basic project work.

### 🖥️ System Administration — 2 Months
Windows Server, Active Directory, Group Policy, backup systems, VMware/KVM virtualization. For those going into IT admin and infrastructure roles.

## How I Teach

Every topic gets two things — explanation, then practice. Not "here's a slide, remember this for the test." We use real cloud environments, real Linux servers, and tools that actual companies use.

Batches are small (max 20 students). I'd rather teach 15 people properly than rush 50 through a syllabus.

Sessions are recorded. If you miss a class or want to rewatch something, it's there. Lifetime access, not just for the course duration.

## About This Repo

Static website — HTML, CSS, vanilla JS. No frameworks, no build tools, no dependencies to manage.

Deployed automatically to GitHub Pages via GitHub Actions every time I push to `main`. The workflow also runs an SEO and HTML quality check before deploying.

```
tech-share/
├── index.html          # Main website
├── style.css           # All styles (dark navy theme)
├── script.js           # Interactivity — terminal animation, FAQ, forms
├── brochure.html       # Downloadable course brochure (print-ready)
├── sitemap.xml         # For Google Search Console
├── robots.txt          # Search engine crawl rules
└── .github/
    └── workflows/
        ├── deploy.yml  # CI/CD → auto deploy to GitHub Pages
        └── quality.yml # HTML audit + SEO checks on every push
```
